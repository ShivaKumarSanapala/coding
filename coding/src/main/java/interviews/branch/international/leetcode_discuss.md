**Round 1: Online Assessment** (75 minutes)

Q1: Priority queue based question (easy-medium)
Q2: Bit manipulation, efficient precomputation of factorial and inverse alongwith combinatorics to pass all testcases (medium-hard)

Solved both questions fully.

---

**Loop Interviews** : All rounds happened on the same day a week after giving OA.

**Round 2: Problem Solving** (DSA) (60 minutes)

Interviewer mentioned 2 questions are to be solved in this round. Expectation was to write production ready code and provide testcases as well. However, I could solve only 1 question before time ran out.

Q: Given a string and a vector of string of random `[start,end]` positions, we need to place bold tags: `<b> at start` and `</b> at end`.

https://leetcode.com/problems/add-bold-tag-in-string/description/

Handled overlapping intervals by merging them. Implemented end to end optimal code and came up with testcases other than those which interviewer provided. Discussed some edge cases and code worked for all scenarios. Discussed TC and SC.

Interviewer provided no hints/suggestions and was silent throughout.

[Self Verdict: Lean Hire]

**Round 3: Problem Solving** (DSA) (60 minutes)

Q1: https://leetcode.com/problems/single-element-in-a-sorted-array/description/

Discussed brute force and optimal approach in detail with interviewer with dry run. Once we both agreed to the solution coded up working solution. Wrote testcases and discussed minor improvements for edge cases. Discussed TC.

Q2: https://leetcode.com/problems/largest-number/description/

Discussed lexicographical approach then interviewer asked to think of several testcases and dry run them. Figured out the problem with my initial approach and discussed optimal approach with merge sort and heap sort. Wrote the recursive call tree for a sample case. Discussed TC and recursive call SC. Coding was not asked for this question.

Q3: Explain BFS and DFS and discuss when to use.

Discussed the traversal algorithms and data structures for implementation. Discussed about infinite branch problem of DFS. Discussed custom traversal approach based on some heuristic combining BFS and DFS for faster convergence like IDDFS. Also went through real-world use cases of such algorithms like BFS for social network analysis.

Overall had lots of insightful discussion and interviewer was super friendly and helpful. Discussed the impact and use cases of AI for a SWE at Microsoft.

[Self Verdict: Strong Hire]

**Round 4: AA | Hiring Manager** (Behavioural+System Design) (60 minutes)

Interviewer started with asking some behavioural questions regarding why thinking of job change and why Microsoft. Discussed about some areas of interest and past work experience.

Q: Low Level Design of Snake and Ladder Game
Requirements:
R1: Board has NxM positions, cannot go beyond boundaries.
R2: K players each rolling dice in round robin format.
R3: Implement snakes and ladders. Both cannot be at same position.
R4: Player needs to roll a 6 to start.
R5: Rolling a 6 gives second chance to roll dice.
R6: Three consecutive 6 means chance cancelled.
R7: If roll is invalid turn goes over to next player.
R8: Game concludes when any player reaches last position.

Implemented all the requirements. Wrote structured classes, functions and data structures for players, position, board, and gameplay. Discussed some optimizations of time and space and implemented them. Interviewer pointed out some edge cases and made required edits.

Had some time leftover and interviewer was in Copilot team so got a chance to discuss the latest developments of AI and LLM happening inside Microsoft and what might be coming next.

[Self Verdict: Strong Hire]

---

**Final Verdict** üéâ

Got verbal offer and document submission mail within 3 working days. Written offer was released after another week.

Overall the process was smooth and very fast. Interviewers and recruiters were prompt and helpful.

```
class Solution {
    public List<Integer> preorderTraversalIterative(TreeNode root,List<Integer> ans){
        if(root==null) return ans;
        Stack<TreeNode> s = new Stack<TreeNode>();
        s.push(root);
        while(!s.isEmpty()){
            TreeNode node = s.pop();
            ans.add(node.val);

            if(node.left!=null) s.push(node.left);
        }
        return ans;
```

```
class Solution {

    // Iterative function for inorder tree traversal
    static ArrayList<Integer> inOrder(Node root) {
        ArrayList<Integer> ans = new ArrayList<>();
        Stack<Node> s = new Stack<>();
        Node curr = root;

        while (curr != null || !s.isEmpty()) {

            // Reach the left most Node of the curr Node
            while (curr != null) {

                // Place pointer to a tree node on
                // the stack before traversing
                // the node's left subtree
                s.push(curr);
                curr = curr.left;
            }

            // Current must be NULL at this point
            curr = s.pop();
            ans.add(curr.data);

            // we have visited the node and its
            // left subtree. Now, it's right
            // subtree's turn
            curr = curr.right;
        }

        return ans;
    }
```

- DFS [Stack]
    - Explores as deeply as possible along each branch before backtracking
    - In the DFS Recursive and Iterative ways, Mark the startNode as True in the Visited Array before iterating a loop OR before calling recursive function

```
    import java.util.Stack;
    import java.util.List;
    import java.util.ArrayList;
 
    class Graph {
        private int V;
        private List<List<Integer>> adj;
 
        public Graph(int V) {
            this.V = V;
            adj = new ArrayList<>(V);
            for (int i = 0; i < V; i++) {
                adj.add(new ArrayList<>());
            }
        }
 
        public void addEdge(int u, int v) {
            adj.get(u).add(v);
        }
 
        public void DFS_Iterative(int startNode) {
            boolean[] visited = new boolean[V];
            Stack<Integer> stack = new Stack<>();
 
            stack.push(startNode);
            visited[startNode] = true;
 
            while (!stack.isEmpty()) {
                int currentNode = stack.pop();
                if(!visited[currentNode]){
                    System.out.print(currentNode + " ");
    
                    for (int neighbor : adj.get(currentNode)) {
                        if (!visited[neighbor]) {
                            visited[neighbor] = true;
                            stack.push(neighbor);
                        }
                    }
                }
            }
        }
    }
```

```
    class Graph { 
        public void BFS_Iterative(int startNode) {

            queue.add(startNode);
            visited[startNode] = true;

                int currentNode = queue.poll();

                    if (!visited[neighbor]) {
                        visited[neighbor] = true;
                        queue.add(neighbor);

                }
```

# **Topological Sort**
- Topological sorting for Directed Acyclic Graph (DAG) is a linear ordering of vertices such that for every directed edge u->v, vertex u comes before v in the ordering.
- Note: Topological Sorting for a graph is not possible if the graph is not a DAG.

- DFS way, (TC - O(V + E) & SC - O(V))
- Idea: node/vertex that comes at last must be present at last. Hence, the idea is to store last visited at bottom. Thus, Stack comes into picture.
- Algorithm,
    - Make visited Array to tackle both disconnected & visited nodes.
    - Call DFS on each unvisited node
        - Call DFS on unvisited Neighbours
        - After making all calls to Neighbours Store current Node in a Stack.
    - Finally, pop all the elements of stack into resultant Vector
```
class Solution {
    public static void recursiveDFS(int v, List<Integer>[] adj, boolean[] visited, Stack<Integer> stack) {
        visited[v] = true;
        for (int i : adj[v]) {
            if (!visited[i]) {
                recursiveDFS(i, adj, visited, stack);
            }
        }
        stack.push(v);
    }

    public static ArrayList<Integer> topoSort(int V, int[][] edges) {
        boolean[] visited = new boolean[V];
        Stack<Integer> stack = new Stack<>();
        List<Integer>[] adj = new ArrayList[V];
        ArrayList<Integer> ans = new ArrayList<>();

        //adding all the vertices
        for (int i = 0; i < V; i++) {
            adj[i] = new ArrayList<>();
        }

        for (int[] edge : edges) {
            adj[edge[0]].add(edge[1]);
        }

        for (int i = 0; i < V; i++) {
            if (!visited[i]) {
                recursiveDFS(i, adj, visited, stack);
            }
        }

        while (!stack.isEmpty()) {
            ans.add(stack.pop());
        }

        return ans;
    }
}
```

- Topological Sort - BFS way (Kahn's Algorithm), (TC - O(V + E) & SC - O(V))
- Idea : The idea is that all nodes which will be at starting will have indegree 0.
- Algorithm,
    - Store Indegree of all nodes in Array.
    - Push nodes in Queue whose indegree == 0.
    - Now, For each node in Queue.
        - Pop the current node & Store into resultant Array
        - Remove indegree count of all neighbours of current node.
        - If neighbours indegree becomes 0 then push into queue.
    - Finally, return resultant Array.
    - Here, Visited Array does not come into the picture.
```
class Solution {
    //function to return list containing vertices in topological order
    public static ArrayList<Integer> topoSort(int V, int[][] edges) {
        ArrayList<ArrayList<Integer>> adj = new ArrayList<ArrayList<Integer>>();
        
        // Initialize adjacency list
        for (int i = 0; i < V; i++) {
            adj.add(new ArrayList<>());
        }

        // Build the adjacency list from edges
        for (int[] edge : edges) {
            adj.get(edge[0]).add(edge[1]);
        }
        
        // Step 1: Calculate in-degree
        int[] indegree = new int[V];
        for(int u = 0; u < V; u++) {
            for(int v : adj.get(u)) {
                indegree[v]++;
            }
        }        

        // Step 2: Add nodes with in-degree 0 to q
        Queue<Integer> q = new LinkedList<>();
        for (int i = 0; i < V; i++) {
            if (indegree[i] == 0) {
                q.offer(i);
            }
        }

        // Step 3: BFS
        ArrayList<Integer> topoOrder = new ArrayList<Integer>();
        while (!q.isEmpty()) {
            int node = q.poll();
            topoOrder.add(node);

            for (int neighbor : adj.get(node)) {
                indegree[neighbor]--;
                if (indegree[neighbor] == 0) {
                    q.offer(neighbor);
                }
            }
        }

        // Optional: Check if topological sort is possible (i.e., no cycle)
        if (topoOrder.size() != V) {
            System.out.println("Cycle detected! Topological sort not possible.");
            return new ArrayList<>();
        }

        return topoOrder; //(Note: Multiple valid topological orders may exist)
    }
}
```

- Bellman-Ford [DP] - Single Source Shortest Path (SSSP) - This can handle Negative Edge Weight and Negative Weight Cycle - Works on both directed and undirected graph with non-negative edge weights as well as directed graph with negative edge weights as long as no negative cycles are present.
- The Bellman-Ford algorithm can detect negative weight cycles, but it cannot find shortest paths in a graph containing them. While it can handle negative edge weights, if a negative cycle is present, the algorithm will continue to find shorter paths indefinitely as it loops through the cycle, indicating the absence of a true shortest path.
    - TC - O(V*E) and SC - O(V)
```
//Back-end complete function Template for Java
class Solution {
    public int[] bellmanFord(int V, int[][] edges, int src) {

        // Initially distance from source to all other vertices
        // is not known(Infinite).
        int[] dist = new int[V];
        Arrays.fill(dist, (int)1e8);
        dist[src] = 0;

        // V X E
        // Relaxing an Edge means Try to Improve the shortest path
            // estimate to a node
        // Relaxation of all the edges V times, not (V - 1) as we
        // need one additional relaxation to detect negative cycle
        // If Graph contain V vertices, it should have V-1 Edges.
        // If V edges, it could form the Cycle.
        for (int i = 0; i < V; i++) {
            for (int[] edge : edges) {
                int u = edge[0];
                int v = edge[1];
                int wt = edge[2];
                if (dist[u] != 1e8 && dist[u] + wt < dist[v]) {

                    // Vth relaxation to check negative cycle
                    // If this is the Vth relaxation, then there is
                    // a negative cycle
                    if (i == V - 1) return new int[] {-1};

                    // Update shortest distance to node v
                    dist[v] = dist[u] + wt;
                }
            }
        }
        return dist;
    }
}
```

```
class Solution {
    static int kruskalsMST(int V, int[][] edges) {
        // Sort all edges based on weight
        Arrays.sort(edges, Comparator.comparingInt(e -> e[2]));

class DSU {
    private int[] parent, rank;

    public DSU(int n) {
        parent = new int[n];
        rank = new int[n];
        for (int i = 0; i < n; i++) {
            parent[i] = i;
            rank[i] = 1;
        }
    }

    public int find(int i) {
        if (parent[i] != i) {
            parent[i] = find(parent[i]);
        }
        return parent[i];
    }

    public void union(int x, int y) {
        int s1 = find(x);
        int s2 = find(y);
        if (s1 != s2) {
            if (rank[s1] < rank[s2]) {
                parent[s1] = s2;
            } else if (rank[s1] > rank[s2]) {
                parent[s2] = s1;
            } else {
                parent[s2] = s1;
                rank[s1]++;
            }
        }
    }
}
```

## What DFS means ?:
- DFS stands for `Depth-First Search`, where "Depth" means we keep going deeper into each child node as long as there are children exploring down a path before backtracking.
  It‚Äôs like diving deep into one branch of the tree before checking the others.
---
## very Simple Example:
- if we have a graph from `1 to 7` Like This One:
```

       (1)
      /   \
     v     v
   (2)     (3)
    |      / \
    v     v   v
   (4)   (5) (6)
            |
            v
          (7)
```
---

## Let's see how DFS works:

- First, we mark **Node(1)** as visited ‚Äî that's our starting point.
- Then, we assume we follow the left path to **Node(2)** ‚Üí from there to **Node(4)**.
- Now we ask: **Did we visit all the nodes?** Nope. So we backtrack.
- Is **Node(2)** already visited? Yes ‚Üí back again to **Node(1)**.
- Try the other path ‚Üí **Node(3)** ‚Üí then **Node(5)** ‚Üí then **Node(7)**.
- After that, backtrack to **Node(3)** again and try the last child ‚Üí **Node(6)**.
- We use a stack to track this path.

---

## DFS with Stack (Simulation):

| Step | Stack                  | Action                                               |
|------|------------------------|------------------------------------------------------|
| 1    | [1]                    | Start at Node(1)                                     |
| 2    | [1, 2]                 | Move to Node(2)                                      |
| 3    | [1, 2, 4]              | Move to Node(4)                                      |
| 4    | [1, 2]                 | Node(4) has no children ‚Üí pop                        |
| 5    | [1]                    | Backtrack to Node(2) ‚Üí already visited ‚Üí pop         |
| 6    | [1, 3]                 | Go to second path ‚Üí Node(3)                          |
| 7    | [1, 3, 5]              | Move to Node(5)                                      |
| 8    | [1, 3, 5, 7]           | Move to Node(7)                                      |
| 9    | [1, 3, 5]              | Node(7) done ‚Üí pop                                   |
|10    | [1, 3]                 | Node(5) done ‚Üí pop                                   |
|11    | [1, 3, 6]              | Move to Node(6)                                      |
|12    | [1, 3]                 | Node(6) done ‚Üí pop                                   |
|13    | [1]                    | Node(3) done ‚Üí pop                                   |
|14    | []                     | All nodes visited ‚Üí DFS finished                     |

>  DFS is all about going as deep as possible first, then backtracking step-by-step ‚Äî and **stack** is the perfect structure for that.
